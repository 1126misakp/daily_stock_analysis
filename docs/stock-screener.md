# 自研选股引擎说明

DSA 内置「自研选股引擎」替换了原 AlphaSift 第三方渠道。它复用本项目 `data_provider` 的 Tushare 全市场批量接口取数，与现有 LiteLLM 通道（运行时走 MiMo）做轻量重排，整体两段式、无外部依赖、无需安装。

> 风险提示：选股结果仅用于研究和辅助判断，不构成投资建议；市场有风险，交易决策和损益由使用者自行承担。

## 两段式流程

1. **第 1 段（量化足切）**：在全市场近 60 个交易日快照上用 pandas 向量化跑策略量化信号，足切到候选池（策略池≤80，仅偏好池≤150）。
2. **第 2 段（LLM 轻量重排）**：把候选池摘要交给现有 LiteLLM 通道做一次轻量重排，输出排序 + 理由 + 风险 + 与偏好的契合度。LLM 不可用时自动降级为按量化打分排序。

## 8 个内置策略

策略量化判据取自 `strategies/<id>.yaml`，并新增 `trading_style`（操盘风格）字段用于前端展示「适合：…」：

| 策略 id | 名称 | 操盘风格 |
|---------|------|----------|
| ma_golden_cross | 均线金叉 | 趋势确认、稳健追涨 |
| volume_breakout | 放量突破 | 追涨、激进 |
| bottom_volume | 底部放量 | 抄底、左侧反转 |
| shrink_pullback | 缩量回踩 | 回调低吸、稳健 |
| one_yang_three_yin | 一阳夹三阴 | 趋势延续入场、中性 |
| growth_quality | 成长质量 | 中长线价值、保守 |
| box_oscillation | 箱体震荡 | 高抛低吸、稳健 |
| bull_trend | 默认多头趋势 | 趋势跟随、中性偏进取 |

## 策略与用户偏好

- **策略与偏好至少填写一个**，否则报错。
- **仅策略**：按策略量化足切 → LLM 重排。
- **仅偏好**：必须能从偏好文本识别出板块（行业子串匹配），按行业 + 基础流动性过滤后量价粗排 → LLM 重排；无法识别板块时提示补充。
- **策略 + 偏好**：第 1 段不按偏好硬过滤（避免命中行业与偏好板块不相交时清空候选），偏好在第 2 段 LLM 重排阶段优先满足（可少选、宁缺毋滥）。

## API 端点

- `GET /api/v1/screen/strategies`：列出 8 个策略（含 `trading_style`）。
- `POST /api/v1/screen/jobs`：提交异步选股 job（`strategy` / `preference` / `max_results`），返回 `jobId`。
- `GET /api/v1/screen/jobs/{job_id}`：轮询 job 状态与结果（内存存储、不持久化，单 worker、进行中幂等复用、TTL 1 小时）。

选股为全市场扫描的耗时操作，采用异步 job + 前端轮询，规避前置 CDN 的 100s 超时上限。

## 数据源

仅在选股流程内通过 Tushare 全市场批量接口（按交易日）取数，属独立调用路径，**不修改 `data_provider` 的路由/优先级/fetcher 链**，不影响实时行情与日 K 的数据源优先体系。
